# References 索引

这些文件不是默认全量加载的说明书，而是按需读取的补充判断文档。

## 什么时候读哪一份

### `input-schema.md`
在以下情况读取：
- 用户没有直接给 JSON，而是给自然语言需求
- 你不确定是图片任务还是视频任务
- 你不确定最小可运行输入是否已经足够
- 你在构造输入前，不确定哪些字段必须补齐
- 你担心会臆造品牌色、参考图、参考视频或 URL

### `scenes.md`
在以下情况读取：
- 你不确定该选 `wechat_cover` 还是 `xiaohongshu_cover`
- 你怀疑用户要的是重排版，而不是从零出图
- 用户需求同时出现平台词和改版词
- 你需要处理图片 scene 冲突

### `api-behavior.md`
在以下情况读取：
- 你要确认 `config.yaml` 应该怎么写
- 你要判断 API、输出文件、`.prompt.txt` 或返回 JSON 是否正常
- 你要排查为什么没出图、没出视频、没有 `video_url`
- 你要确认 Wan 2.6 / 2.7 视频协议差异

## 不要全读的场景

- 只是标准图片生成，不要把视频排错说明也一起加载
- 只是标准视频生成，不要把 `scenes.md` 当成视频路由文档
- 用户已经给了完整 JSON，不要为了保险重读 `input-schema.md`
- 只是确认 scene，不要顺手把 API 文档也一起读掉

## 使用原则

- 先用 `SKILL.md` 做主路由判断
- 只有在主文件信息不够时，再加载对应 reference
- 不要把 reference 当成教程从头到尾通读
